# 設計書

## 1. 概要

- 対象プロダクト
- 目的 / ゴール
- スコープ（対象・対象外）

## 2. 前提・参照

- 要件定義書
- 技術スタック
- 非機能要件

## 3. アーキテクチャ

- 全体構成（クライアント / Firebase 各サービス）
- 認証フロー概要
- データフロー概要
- 通知フロー概要

## 4. データモデル（Firestore）

- コレクション一覧
- 主要フィールド
- リレーション/参照
- インデックス（必要な検索）
- データ保持/削除ポリシー

## 5. 権限・セキュリティ

- 役割と権限（管理者/子供）
- Security Rules 方針
- 招待コード/招待リンクの扱い
- 退会時のデータ処理

## 6. 画面設計

- 画面一覧
- 画面遷移
- 主要ユーザーフロー（投稿/コメント/招待/退会 など）
- 状態管理の粒度

## 7. 機能設計

- 認証
- ファミリー
- ファミリーメンバー
- 掲示板/投稿/コメント
- 画像添付
- 既読/ピン留め
- 通知
- チュートリアル

## 8. 非機能要件への対応

- パフォーマンス
- 可用性/オフライン
- セキュリティ
- アクセシビリティ
- ログ/分析

## 9. テスト戦略（TDD）

- 単体テスト
- 統合テスト
- E2E テスト
- モック/スタブ方針

## 10. 前提/未確定事項

- 追加確認が必要な点
